## What's Changed

### New Features
- Added Markdown-to-PDF rendering through Pandoc-generated HTML and WeasyPrint, including `md to-pdf`, external-tool readiness checks, page presets, ToC controls, asset policy validation, and template materialization.
- Added reusable Markdown PDF profiles through `md pdf-profile init`, with YAML/JSON profile input, cover metadata, page chrome, page-number controls, CLI metadata overrides, and deterministic option precedence.
- Added mixed-language font support for Markdown PDF profiles, plus `font list`, `font inspect`, and `font check` commands for discovery, diagnostics, glyph coverage checks, TTC face metadata, and fontconfig-backed coverage when available.
- Added opt-in Shiki code highlighting for Markdown PDFs, including bundled language support, line numbers, and transformer notation for highlighted, diff, focus, error, and warning lines.

### Improvements
- Tightened Markdown PDF asset URL validation so local and remote asset handling remains explicit and predictable.
- Refined PDF profile and template preset behavior, including clearer public guidance for profile persistence and default preset materialization.
- Updated public font naming to use user-facing Google Fonts family names such as `Noto Sans TC` instead of internal `CJK` suffix wording.
- Split large CLI, rename, data-query, Markdown, XLSX, and test modules into focused files without changing the intended command behavior.

### Bug Fixes
- Supported bundled Shiki languages in Markdown PDF highlighting so configured language aliases resolve consistently.
- Tightened PDF profile normalization types and restored a clean TypeScript no-emit check.
- Corrected the capitalization of "Pandoc" in the DOCX conversion description.

### Compatibility
- Codex SDK baseline for `v0.1.3`: `0.130.0`
- Markdown PDF profile and template workflows are direct CLI flows in this release. Interactive Markdown PDF and Codex-assisted PDF helper flows are deferred to a later release.
